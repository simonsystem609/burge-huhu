# Security Policy

## Supported version

Security fixes are considered for the current `master` branch and the version
deployed from it. Older commits and unofficial deployments may not receive
security updates.

## Reporting a vulnerability

Use this repository's **Security** tab and submit a report through GitHub
Private Vulnerability Reporting. Please do not open a public issue for a
suspected vulnerability.

Include the affected commit or deployment, the expected impact, and concise
reproduction steps. Do not include credentials, private data, or information
about other users.

Reports are reviewed on a best-effort basis. Please allow time to investigate
and prepare a coordinated fix before publishing details.

## Safe testing

Reproduce issues locally where possible. Do not perform load testing,
denial-of-service testing, automated scanning, or disruptive gameplay against
the public Render deployment.
